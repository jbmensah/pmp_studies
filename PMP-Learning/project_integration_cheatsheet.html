9. Change Requests & Change Management Plan

â™»ï¸ Formal Change Control

Change requests are formal proposals to modify a baseline or configuration item. The change management plan defines how those changes are processed.

Types of Change Requests
  • Corrective action: brings performance back in line with the plan.
  • Preventive action: prevents future deviations.
  • Defect repair: fixes non-conforming deliverables.
  • Updates: changes to plans or documents.
Change Management Plan
  • Defines who can raise change requests (any stakeholder).
  • Specifies impact analysis (cost, schedule, scope, risk).
  • Identifies the approval authority (PM, sponsor, CCB).
  • Requires outcomes to be documented in a change log.
  • Ensures requesters are informed of decisions (approved / deferred / rejected).
ğŸ§­ Exam hint: When a stakeholder asks for a change that impacts scope, schedule, or cost: do not implement immediately. First: check the plan, analyze impact, submit a formal change request.

An issue log is used to capture, track, and manage problems, conflicts, or gaps that need attention during the project.
