1. Cost Management ‚Äî Key Concepts

üí° Big Picture

Cost Management is about planning, estimating, budgeting, financing, funding and controlling project costs so the project can be completed within the approved budget.

Core Ideas
  • Costs start as rough estimates and become more accurate over time.
  • We add contingency and possibly management reserves to handle uncertainty.
  • We monitor performance using Earned Value Analysis (EVA).
Key Topics
  • Estimate ranges: ROM, budget, definitive.
  • Contingency vs management reserves.
  • Accuracy vs precision of estimates.
  • Earned Value and variance analysis.
Why It Matters
  • Executives care about budget predictability.
  • Helps avoid late surprises and funding gaps.
  • Supports early decisions on feasibility and trade-offs.
üß≠ Key takeaway: Cost management is not just ‚Äúmaking a budget‚Äù ‚Äî it is about progressively refining estimates and actively controlling performance with reserves and metrics.

2. Estimate Ranges

üìä Estimate Types

Cost estimates become more accurate as the project moves from initiation to planning to execution. Early on, uncertainty is high, so we use ranges instead of single numbers.

Typical Estimate Types & Ranges
Type When Used Typical Range
Rough Order of Magnitude (ROM) Very early (initiation / feasibility) ‚âà -25% to +75%
Preliminary Early planning / concept ‚âà -15% to +50%
Budget Estimate Planning, for overall budget ‚âà -10% to +25%
Definitive Estimate Late planning / near execution ‚âà -5% to +10%
Final / As-built After work is done 0% (actual)

These specific ranges might not be tested numerically on the exam anymore, but they are very useful in real projects for setting expectations with sponsors.

How Estimates Are Built
  • Top-down (analogous): use similar past projects; faster, less accurate.
  • Bottom-up: estimate each work package and roll-up; slower, more accurate.

Early = ROM / analogous. Later = definitive / bottom-up, based on detailed WBS and real information.

üß≠ Exam hint: ‚ÄúEarly in the project with limited info‚Äù ‚Üí think ROM / analogous / wider range. ‚ÄúLater with detailed scope and WBS‚Äù ‚Üí think definitive / bottom-up.

3. Contingency vs Management Reserves

üìë Reserves

Reserves are extra money in the budget to deal with uncertainty, but they serve different purposes and sit in different places in the budget structure.

Contingency Reserve
  • Set aside for identified risks in the risk register.
  • Based on risk impact analysis (e.g., probability √ó impact).
  • Added on top of work package cost estimates.
  • Included in the Cost Baseline.
  • Used when known risks occur (‚Äúknown-unknowns‚Äù).
Management Reserve
  • Set aside for unforeseen work within scope.
  • Not tied to specific identified risks.
  • Not part of the cost baseline, but part of the total project budget.
  • Typically controlled by sponsor, PMO, or senior management.
  • Used when scope-related surprises appear but are still considered in-scope.
üß≠ Exam hint: Contingency = for identified risks, inside cost baseline. Management reserve = for unknowns, outside baseline, part of total budget.

4. Accuracy vs Precision

üí° Estimate Quality

Accuracy and precision are not the same thing. The exam loves to test this distinction.

Accuracy
  • Relates to correctness of the estimate.
  • Often expressed as the range around an estimate.
  • Low accuracy ‚Üí wide range (e.g., -25% to +75%).
  • High accuracy ‚Üí narrow range (e.g., -5% to +10%).
  • Accuracy improves as the project progresses and more is known.
Precision
  • Relates to exactness of the value.
  • Example: 995.30 is more precise than 1,000.
  • ‚ÄúTwo days‚Äù is more precise than ‚Äúnext week‚Äù.
  • Precision is about the number of decimal places / level of detail, not about being correct.
üß≠ Exam hint: An estimate can be precise but inaccurate (e.g., 99,532.17 but far from reality) or accurate but not very precise (e.g., ‚Äú~100k, ¬±5%‚Äù).

5. Earned Value Analysis ‚Äî Core Concepts

üìä Performance Measures

Earned Value Analysis (EVA) compares the plan to the actual performance in terms of both cost and schedule. It tells you if you are ahead/behind schedule and under/over budget.

Core Terms
  • PV ‚Äî Planned Value: value of work that should be done by now.
  • EV ‚Äî Earned Value: value of work that is actually completed now.
  • AC ‚Äî Actual Cost: money actually spent so far.
  • BAC ‚Äî Budget at Completion: total planned budget for the project.
Interpreting the Basics
  • If EV < PV ‚Üí you have done less work than planned ‚Üí behind schedule.
  • If AC > EV ‚Üí you spent more than the value delivered ‚Üí over budget.
  • If EV > AC ‚Üí you delivered more value than cost ‚Üí under budget at this point.
üß≠ Key idea: EV is always the central figure ‚Äî it is the ‚Äúvalue of work actually completed‚Äù. PV and AC are compared to EV to get schedule and cost signals.

6. Variance Analysis & Forecasting (EAC)

üìä Variances & Indices

Using PV, EV and AC, we can calculate variances and indices to understand how well we are performing and to forecast the final cost.

Cost & Schedule Variance
  • Cost Variance (CV) = EV ‚àí AC
    • CV < 0 ‚Üí over budget.
    • CV > 0 ‚Üí under budget.
  • Schedule Variance (SV) = EV ‚àí PV
    • SV < 0 ‚Üí behind schedule.
    • SV > 0 ‚Üí ahead of schedule.
Performance Indices
  • Cost Performance Index (CPI) = EV / AC
    • CPI < 1 ‚Üí cost efficiency is poor (over budget).
    • CPI > 1 ‚Üí cost efficiency is good (under budget).
  • Schedule Performance Index (SPI) = EV / PV
    • SPI < 1 ‚Üí behind schedule.
    • SPI > 1 ‚Üí ahead of schedule.
Forecasting ‚Äî Estimate at Completion (EAC)

EAC is the updated forecast of the total project cost, given current performance.

  • Simple EAC (one common form)
    EAC = BAC + AC ‚àí EV
  • Performance-based EAC (using CPI)
    EAC = BAC / CPI

Use BAC / CPI when you assume the current cost efficiency will continue. Use BAC + AC ‚àí EV when you expect future work to follow the original plan.

üß≠ Exam hint: Remember the direction: in most EVA formulas, AC is in the denominator or subtracted, and indices are centered around 1 (1 = on target).

üî• Exam-Focused Summary ‚Äî Cost Management

Concept Flow
  • Start with rough estimates (ROM) ‚Üí refine to budget and definitive estimates.
  • Add contingency (for identified risks) to build the cost baseline.
  • Optionally add management reserve for unknowns to get the total budget.
  • Track progress using PV, EV, AC.
  • Calculate variances and indices (CV, SV, CPI, SPI) to see if you are on budget/on schedule.
  • Update forecasts with EAC when performance patterns emerge.
What the Exam Cares About
  • Knowing the difference between estimate types and when to use each.
  • Clearly distinguishing contingency vs management reserves.
  • Understanding accuracy vs precision in estimates.
  • Interpreting CPI, SPI, CV, SV (over/under budget, ahead/behind schedule).
  • Choosing the right EAC formula based on the scenario description.
üéì One-liner: Cost Management is about accepting uncertainty, adding smart reserves, and using earned value to see early if you are in trouble so you can adjust before it is too late.
