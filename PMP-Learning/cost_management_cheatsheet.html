  • Reserves — contingency & management reserves added to deal with risk and unknowns.

  • Earned Value Analysis (EVA) — classic technique to compare plan vs actual cost & schedule performance.
For the exam, focus on: different estimate ranges, the purpose of each reserve, and the “< 1 / > 1” logic for CPI & SPI.
2. Estimate Ranges
How estimate accuracy improves over time

Early in the project, cost estimates have a wide range (low accuracy). As we move into planning and execution, more detailed information allows tighter, more accurate estimates.

Estimate Type Typical Timing Accuracy Range (approx.) How it’s Built
Rough Order of Magnitude (ROM) Very early / initiation, discovery −25% to +75% Top-down, analogous, minimal detail. “Ballpark” only.
Preliminary Concept stage −15% to +50% Refined ROM, still high-level, often for early approvals.
Budget estimate Early planning −10% to +25% More breakdown; used for initial budget baselines.
Definitive Detailed planning −5% to +10% Bottom-up from work packages; most accurate planned estimate.
Final / As-built After work is completed 0% (actual cost) Real cost once scope has been delivered.

Bottom-up estimating takes longest but is the most accurate, and is typically used in the detailed planning phase once the WBS/work packages are clear.

3. Contingency & Management Reserves
Two different “safety nets” in your budget
Contingency Reserve
  • Money set aside for identified risks in the risk register.
  • Added to the work package cost estimates → forms the cost baseline.
  • Used when those known risks actually occur.
For known-unknowns
Management Reserve
  • Extra budget for unforeseen scope-related work.
  • Not part of the cost baseline; sits on top of it.
  • Controlled by sponsor / PMO / sometimes PM; released through change control.
For unknown-unknowns
Exam logic: Cost baseline = estimates + contingency. Total project budget = cost baseline + management reserves.
4. Accuracy vs Precision
Same number? Not always the same meaning.

Accuracy and precision are easy to mix up. In cost questions, they usually relate to:

Accuracy — “How correct?”
  • Refers to closeness to the true value, expressed as a range.
  • Low accuracy → wide range (e.g. ROM −25% / +75%).
  • High accuracy → tight range (e.g. definitive −5% / +10%).
Think: size of the uncertainty band
Precision — “How detailed?”
  • Refers to the level of exactness in the number given.
  • “USD 995.30” is more precise than “about 1,000”.
  • “Next Wednesday” is more precise than “sometime next week”.
Think: number of decimal places / specificity
Typical trap: a number can look very precise (e.g. 995.37) but still be inaccurate if it was based on weak data or a very early estimate.
5. Earned Value Analysis (EVA)
Comparing plan vs actual cost & schedule

EVA looks at three core numbers for each work package / project checkpoint:

  • Planned Value (PV) — how much work should be done by now.
  • Earned Value (EV) — value of work actually completed.
  • Actual Cost (AC) — what we have actually spent.
BAC = Budget at Completion (total planned cost) PV grows over time until it equals BAC at project finish

Visual signals: if AC is above PV, you’re spending faster than planned. If EV is below PV, you’ve delivered less work than planned.

6. Variance Analysis & Forecasting
Using EVA numbers to understand performance

Variance analysis combines EV, PV, and AC to show whether you are on or off track.

Cost performance
CV (Cost Variance) = EV − AC
  • CV > 0 → under budget (good).
  • CV < 0 → over budget (bad).
CPI (Cost Performance Index) = EV ÷ AC
  • CPI > 1 → getting more value per dollar (cost efficient).
  • CPI < 1 → poor cost efficiency, overspending.
Schedule performance
SV (Schedule Variance) = EV − PV
  • SV > 0 → ahead of schedule.
  • SV < 0 → behind schedule.
SPI (Schedule Performance Index) = EV ÷ PV
  • SPI > 1 → delivering faster than planned.
  • SPI < 1 → slower than planned.
Forecasting — Estimate at Completion (EAC)

BAC = original Budget at Completion. EAC adjusts this based on actual performance.

  • Simple adjustment (one-time variance):
    EAC = BAC + AC − EV
  • Performance-based (current CPI continues):
    EAC = BAC ÷ CPI
Shortcut memory: in cost formulas, AC is always the “troublemaker” — it’s the term you subtract from or divide into. CPI & SPI both compare EV against AC or PV and are centered around 1.0.
