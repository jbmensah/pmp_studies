üöÄ Introduction & Core Concepts

Cost management is about estimating, setting a time-phased cost baseline, and then controlling via performance data (especially EVM). On the exam, expect ‚Äúwhat do these numbers mean?‚Äù more than math.

Plan-Driven (Traditional) üß±

  • WBS ‚Üí Work Packages
  • Activities build the work packages

  • Backlog ‚âà WBS (features ‚Üí stories ‚Üí tasks)
  • Stable team cost per iteration (burn rate)

Reserves Distinction üõë

Contingency (Known-Unknowns)

For identified risks. Included in the cost baseline. PM can use within defined thresholds.

Management Reserve (Unknown-Unknowns)

Not in the cost baseline. To use it, you typically need a formal change request to move funds into the baseline.

‚öôÔ∏è Monitoring & Controlling: Control Costs & Resources

After baselines are set, you monitor performance and manage variances. PMI logic: measure ‚Üí analyze ‚Üí act ‚Üí (if baseline impacted) raise change request.

Control Costs: Typical Activities

  • Compare actuals vs baseline; identify variance trends.
  • Use EVM + forecasting (EAC/ETC).
  • Perform reserve analysis for risk.
  • Recommend corrective/preventive actions.
  • Issue change requests when baseline needs adjustment.
üïµÔ∏è Detective mindset: The best exam answers sound like root-cause + data-driven action, not panic actions (‚Äúadd overtime immediately‚Äù) unless justified.

Reserve Analysis in Control Costs

  • Removing contingency (risk retired) ‚Üí typically needs formal approval.
  • Adding contingency (new risk) ‚Üí formal approval.
  • Using management reserve ‚Üí formal approval to move it into baseline.

  4. If baseline impacted ‚Üí change request.

üìä EVM: Metrics & Analysis (Earned Value Management)

EVM integrates scope + schedule + cost to evaluate performance against the baseline and forecast outcomes. Memorize definitions, then master interpretation.

üß† Exam focus: ‚ÄúWhat does CPI/SPI mean?‚Äù beats ‚Äúcalculate CPI/SPI‚Äù. If you can interpret fast, you‚Äôll score fast.

Core Metrics

Metric Acronym Interpretation (As of Today...) Direction
Planned Value PV Value of work planned to be done. Planned
Earned Value EV Value of work actually accomplished. Performance
Actual Cost AC Actual cost incurred for the work accomplished. Spending
Budget at Completion BAC Total budget for the full project (baseline total). Baseline total
Estimate at Completion EAC Forecasted total cost at finish (based on performance). Forecast
Estimate to Complete ETC Forecasted cost to finish remaining work from now. Forecast remaining

Fast Formulas (high yield)

CV = EV ‚àí AC
SV = EV ‚àí PV
CPI = EV / AC
SPI = EV / PV
EAC ‚âà BAC / CPI
ETC = EAC ‚àí AC
VAC = BAC ‚àí EAC
‚úÖ Interpretation cheat:
  • CPI > 1 good (under budget); CPI < 1 bad (over budget)
  • SPI > 1 good (ahead); SPI < 1 bad (behind)
  • CV positive good; SV positive good

Examples (interpretation)

  • If CPI = 1.2 ‚Üí under budget: you‚Äôre getting $1.20 value per $1 spent.
  • If SPI = 0.89 ‚Üí behind schedule: progressing at 89% of planned rate.

üèÉ Agile Budgeting Focus

Agile commonly fixes time + cost, and adjusts scope. Estimation is relative; forecasting uses flow metrics.

Agile metric

Velocity

Work completed per iteration (story points). Helps forecast schedule and budget risk.

Agile metric

Burn Rate

Money spent per iteration (often stable). Combine with velocity to forecast cost.

Agile artifact

Backlog

Prioritized work list (‚âà WBS). Track delivery vs plan using completion trends.

  • Prefer ‚Äúre-prioritize backlog / adjust scope‚Äù over ‚Äúincrease budget‚Äù as first move.
  • Use stable teams ‚Üí predictable costs ‚Üí forecast with burn rate.
