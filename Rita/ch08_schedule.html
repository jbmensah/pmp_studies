Develop Schedule

‚ÄúFind critical path ‚Üí set baseline.‚Äù

Process Group: Planning CPM lives here Output: Schedule Baseline

üì• Key Inputs

  • Network diagram
  • Duration estimates + basis
  • Resource requirements + calendars

üõ†Ô∏è Key Tools

  • Critical Path Method (CPM)
  • Compression: Crashing / Fast-tracking
  • What-if & simulation (Monte Carlo)
  • Resource optimization: leveling/smoothing

üì§ Key Outputs

  • Project schedule
  • Schedule baseline
  • Schedule data; change requests

Float types

  • Total float: slip without impacting project end date
  • Free float: slip without impacting successor early start
  • Project float: slip without missing externally imposed date

Compression options (exam truth)

  • Crashing = adds cost (maybe risk)
  • Fast-tracking = adds risk (often rework)
  • Reduce scope = common in agile (trade scope for time)
